# Codex

## 目的

Codexを使ったリポジトリ編集、Markdown整理、GitHub Pages修正、memory update自動化のルールをまとめる。

## 重要な前提

- 作業前に既存ファイルを読む
- 既存内容を破壊的に上書きせず、統合する
- 変更後は、確認結果と次の行動を短くまとめる
- 個人情報、APIキー、パスワード、メール本文の生データは保存しない

## よく使う情報

- Markdownドキュメントの作成・整理
- ディレクトリ構成の初期設計
- GitHub Pagesの不具合調査
- Mermaid mindmapの更新
- 会話後の記憶更新案作成
- commit message案の作成

## 次回AIに読ませるべき内容

- Codexに作業を頼むときは [../workflows/codex-task-template.md](../workflows/codex-task-template.md) を使う
- リポジトリ更新後は [../workflows/update-memory.md](../workflows/update-memory.md) を使う
- GitHubへ反映するときは [../workflows/github-push-flow.md](../workflows/github-push-flow.md) を読む

## 更新ルール

- 実際に役立った作業手順だけ残す
- ツールの失敗や注意点は、再発防止できる形で記録する
- 一時的なログや不要な実行結果は残さない
