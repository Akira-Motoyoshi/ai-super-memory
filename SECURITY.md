# Security Policy

このリポジトリは、公開される可能性を前提に扱います。

## 保存しない情報

- APIキー、パスワード、トークン、秘密鍵
- 住所、電話番号、詳細な個人情報
- Zoomリンク、会議URL、非公開メール本文の生データ
- 会話ログ全文や未整理の生データ
- 公開されると不利になる選考情報や企業情報

## 保存してよい情報

- 抽象化した自己紹介や判断基準
- 再利用できるプロジェクト要約
- 面接回答や文章作成の型
- GitHub PagesやCodex作業の再発防止メモ

## 漏えいに気づいたとき

1. 該当ファイルをGitHub上で非公開にするか削除する
2. APIキーやパスワードは即時ローテーションする
3. どのファイルに混ざったかを確認する
4. 必要なら履歴からの削除も検討する
5. `decisions/decision-log.md` に再発防止だけを記録する

## Codexに依頼するときの注意

- 編集前に既存ファイルを読ませる
- 保存しない情報を明記する
- push前に差分確認を依頼する
- 迷う情報は保存せず、要約にとどめる
